# Android Farm — Посібник користувача

Короткий огляд панелі керування для користувача.

## Вхід

1. Відкрийте `/login`
2. Введіть email та пароль
3. Після входу відкриється Dashboard

**Реєстрація** — `/register` (якщо увімкнена адміністратором)

---

## Розділи

### Dashboard (`/`)
- Статус агентів
- Статистика платформ та емуляторів
- Швидкий доступ до платформ

### Platforms (`/platforms`)
- Список платформ (Instagram, YouTube, TikTok тощо)
- Кожна платформа має дії: post, like, login, search тощо
- Вибір емулятора та параметрів для виконання

### Emulators (`/emulators`)
- Список емуляторів
- Статус (active/inactive)
- Управління видимістю, клонування

### Accounts (`/accounts`)
- Соціальні аккаунти для платформ
- Додавання, редагування, прив'язка до емуляторів
- Проксі для аккаунтів

### Queue (`/queue`)
- Черга завдань
- Статуси: pending, processing, completed, failed
- Скасування задач

### History (`/history`)
- Історія виконання дій
- Фільтри, деталі, результати

### Posts & Likes (`/posts`)
- Пости, лайки
- Управління постами для лайків

### Captcha (`/captcha`)
- Робота з капчею

### API Keys (`/api-keys`)
- Створення API ключів для програмного доступу

### Blacklist (`/blacklist`)
- Чорний список (блокування певних задач)

---

## Агенти

У заголовку — **селектор агентів**. Виберіть активного агента для виконання дій. Перемикач **видимості** — показувати агента в API та давати йому задачі.

---

## Важливо

- Усі дані завантажуються з backend після входу
- Без авторизації доступ заборонено
- Backend URL налаштовується в `.env.local` (`NEXT_PUBLIC_BACKEND_URL`)
